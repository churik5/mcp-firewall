# Threat catalogue

This document is generated by hand from the YAML rule packs in
`src/bulwark_mcp/rules/builtin/`. Every rule has:

- a **class** of attack it targets;
- a **source** — the public PoC, paper, or library where the pattern was first
  documented (linked so you can review the original);
- an **FPR note** — false-positive risk, based on quick spot-checking against
  benign text. None of these have been measured against a published corpus
  yet; that's a Week-3 task.

If a rule is firing too eagerly on your traffic, drop it from `rules_dir` (or
override `apply_to: []`) and file an issue with the false-positive sample so
we can tighten the pattern.

## Class: role hijack / instruction override

These are the canonical "ignore previous instructions" family — the attacker
embeds an imperative inside data the agent reads, hoping the model will treat
it as a system instruction. All apply only to **server→client** frames; finding
the same words inside a user request is usually legitimate intent.

| Rule id | What it catches | Source | Score | FPR note |
|---|---|---|---|---|
| `role_hijack.ignore_previous` | "ignore (all) previous/prior/above instructions" + variants | [Simon Willison, Sep 2022](https://simonwillison.net/2022/Sep/12/prompt-injection/) | 0.85 | Low — phrase is rare outside attacks. |
| `role_hijack.disregard_above` | "disregard / forget / abandon everything above" | promptfoo `redteam.harmful.injection` | 0.85 | Low. |
| `role_hijack.you_are_now` | "you are now a/an/the …" role replacement | [garak `probes.promptinject.HijackHateHumans`](https://github.com/leondz/garak) | 0.65 | Medium — phrase is common in benign roleplay; deliberate low score. |
| `role_hijack.new_instructions` | "your new instructions are: …" / "new system prompt" | garak `probes.promptinject` | 0.85 | Low. |
| `role_hijack.act_as_jailbreak` | "act as DAN / AIM / STAN / unrestricted" jailbreak prefix | [jailbreakchat.com](https://www.jailbreakchat.com/) | 0.9 | Very low. |
| `role_hijack.fake_system_tag` | `<\|system\|>`, `<\|im_start\|>`, `<\|admin\|>` etc. — synthetic system tags | [garak probes](https://github.com/leondz/garak/tree/main/garak/probes) | 0.8 | Low — these tags should not appear in tool results. |
| `role_hijack.from_now_on` | "from now on, you will / must / shall …" | [Simon Willison's prompt-injection tag](https://simonwillison.net/tags/prompt-injection/) | 0.7 | Medium — common in tutorials; score reflects ambiguity. |

## Class: data exfiltration

These look for instructions that try to make the agent send/forward secrets,
credentials, or session contents to an attacker-controlled destination — usually
via a markdown image (referrer-based exfil), an HTTP request, or a shell command.

| Rule id | What it catches | Source | Score | FPR note |
|---|---|---|---|---|
| `exfil.send_to_url` | "send/POST/exfiltrate … to https://…" | [PromptArmor: Prompt-Injection Data Exfiltration](https://promptarmor.com/blog/prompt-injection-data-exfiltration) | 0.85 | Medium — could fire on legitimate "POST data to https://api.example.com". Tighten with rule-level allowlist if needed. |
| `exfil.curl_to_remote` | inline `curl -X POST …data` exfil command | garak `probes.exploitation` | 0.85 | Low when in tool-result text; medium in c2s. |
| `exfil.markdown_image_with_query` | markdown image whose URL carries data in query string | [embracethered: data exfiltration via markdown images](https://embracethered.com/blog/posts/2023/data-exfiltration-via-markdown-images-claude/) | 0.9 | Very low — image URLs with `?data=…` are extremely uncommon in benign output. |
| `exfil.credential_keywords` | "read/cat/print … env / .aws/credentials / id_rsa / .ssh/" | [embracethered.com](https://embracethered.com/) | 0.85 | Low. |
| `exfil.dns_beacon` | dig/nslookup against burpcollaborator.net, interact.sh, oastify.com, requestbin.com | OWASP A03 / OOB-exfil techniques | 0.9 | Very low. |

## Class: invisible Unicode

These apply to **both directions**. An agent's response or a user's request that
contains zero-width or tag characters is almost always malicious or accidental
copy-paste from an untrusted source. Tag characters (`U+E0000-U+E007F`) are
particularly nasty because they are invisible to humans yet stripped to
ASCII by tokenisers.

| Rule id | What it catches | Source | Score | FPR note |
|---|---|---|---|---|
| `unicode.zero_width_run` | 3+ zero-width chars in a row | [embracethered: Unicode tags](https://embracethered.com/blog/posts/2024/hiding-and-finding-text-with-unicode-tags/) | 0.7 | Very low — single ZWJ is normal in some scripts, but a run of 3+ is suspicious. |
| `unicode.tag_chars` | Unicode TAG characters U+E0000-U+E007F | [embracethered: Unicode tags](https://embracethered.com/blog/posts/2024/hiding-and-finding-text-with-unicode-tags/) | 0.95 | Practically zero in benign content. |
| `unicode.rtl_override` | RTL/LTR override and similar bidi controls | [trojansource.codes](https://trojansource.codes/) (CVE-2021-42574) | 0.7 | Low — these can appear legitimately in mixed-script text but should be rare in MCP traffic. |
| `unicode.bidi_isolate` | runs of 2+ bidi-isolate characters | trojansource.codes | 0.6 | Medium — some legitimate uses; score reflects that. |

## Class: shell injection (c2s only)

These inspect tool-call **arguments** the agent sends to the MCP server. The
LLM classifier is **not** invoked on c2s frames — these regexes are the only
line of defence on outbound argv.

| Rule id | What it catches | Source | Score | FPR note |
|---|---|---|---|---|
| `shell.rm_rf_root_or_home` | `rm -rf` against `/`, `~`, `$HOME`, or `..` | OWASP A03 Injection | 0.95 | Very low. |
| `shell.curl_pipe_to_shell` | `curl/wget` piped directly into a shell or interpreter | OWASP A06 / [MITRE T1059.004](https://attack.mitre.org/techniques/T1059/004/) | 0.9 | Low. |
| `shell.base64_decoded_pipe` | decoded base64 / large echo blob piped into a shell | [MITRE T1027.013](https://attack.mitre.org/techniques/T1027/013/) | 0.9 | Low. |
| `shell.reverse_shell_classic` | `bash -i >& /dev/tcp/`, `nc -e /bin/sh`, `python -c 'import socket'` | [GTFOBins](https://gtfobins.github.io/) | 0.95 | Very low. |
| `shell.history_clearing` | `history -c`, redirect to history file, unsetting HISTFILE | [MITRE T1070.003](https://attack.mitre.org/techniques/T1070/003/) | 0.7 | Medium — history hygiene is a normal admin task; default policy *warns* rather than blocks. |

## Limitations and known false-negative paths

- **Sub-token obfuscation.** The rules detector is a regex layer. Any attacker
  willing to obfuscate (e.g. "1gnore previous instructions", or splitting words
  across newlines) bypasses every pattern here. The LLM classifier is the
  fallback for that, but a 3 B parameter local model will still miss
  imaginative variations.
- **Multilingual injection.** Patterns are largely English. We rely on the
  classifier's multilingual capability for non-English attacks.
- **Tool descriptions.** v0.2 inspects tool *results*, not tool *descriptions*
  returned by `tools/list`. A malicious MCP server could embed steering inside
  tool descriptions and the agent would read it before any tool call. This is
  on the v0.3 list.
- **Streaming responses.** We wait for a complete newline-delimited frame
  before inspecting. Streaming chunks (e.g. SSE) are not yet supported and are
  forwarded verbatim until the transport-level frame closes.

## Adding your own rules

Rule packs live as YAML in `src/bulwark_mcp/rules/builtin/` (shipped) and
optionally `rules/community/` (gitignored, user-supplied). The schema is

```yaml
rules:
  - id: <namespace.snake_case>          # must be unique across packs
    description: "<one-line human description>"
    pattern: '<Python re-compatible regex; flag inline like (?i)>'
    score: 0.0..1.0
    apply_to: [server_to_client, client_to_server]   # at least one
    source: "<URL or library reference>"             # required for THREATS.md
```

Every rule must carry `source` — it is enforced by `tests/test_detectors_rules.py::test_every_rule_has_a_source` so the catalogue cannot drift.
